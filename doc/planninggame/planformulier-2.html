1. SRU Update voorbeeld script

Voorbeeld script dat input leest en een update request maakt en upload.

Bij de documentatie is een simpel voorbeeld script toegevoegd. Dit script leest van stdin en verpakt de gelezen input in een SRU Update request. Vervolgens wordt dit request naar de server gestuurd en wordt het SRW Response getoond.

2. Ondersteuning meerdere Annotaties

Momenteel wordt alleen 1 annotatie per update ondesteund, dit moet worden aangepast zodat meerdere annotaties per update toegestaan zijn. De foutafhandeling zal zodanig worden aangepast dat de batch stopt bij een foute annotatie. De reeds verwerkte annotaties zijn dan wel goed verwerkt.

De server neemt de child-elementen van de binnenkomende rdf:RDF tag en verpakt deze als losse rdf:RDF elementen. Deze worden vervolgens door de server verwerkt in serie. De identifier wordt uit de rdf:Description of oas:Annotation tag gehaald, waardoor de identifier in het SRU Update request vervalt.

3. URN's resolvebaar maken

URN's moeten resolvebaar worden gemaakt. De URN wordt dan een URI die leidt naar een pagina op de server. bijvoorbeeld: "http://oas.dev.seecr.nl/resolve/.....". Op de pagina wordt de inhoud van de annotatie getoond in RDF+XML.

http://oas.dev.seecr.nl:8000/resolve/urn%3Auuid%3A623601ef-bd16-4b5c-97df-d3b1c41e35e3

4. Bestaande referenties opzoeken

Als URN's al eens eerder zijn gegeven deze hergebruiken. Er mag dan geen inhoud bij zitten anders volgt er een foutmelding. Als de URN gevonden is wordt de gerelateerd informatie hergebruikt.
