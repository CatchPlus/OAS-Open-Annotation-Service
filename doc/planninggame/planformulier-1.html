Planformulier 1 - Open Annotation Service

0. Server inrichten

Het opzetten van een server ten behoeve van software ontwikkeling en het tussentijds testen van de ontwikkelde software.

1. SRU update interface

Simpele SRU update interface waarmee een standaard annotatie mee in het systeem kan worden ingevoerd. Enige check is of het een daadwerkelijke annotatie is. Update en delete is op hele annotatie.

2. Voorbeeld

Ten behoeve van testen en acceptatie is het handig om goede voorbeelden te hebben. Deze gaan we verzamelen. Voorbeelden voor het opbouwen van het systeem maar ook voorbeelden die voldoen aan de verwachting. Deze kunnen dan gebruikt worden ter acceptatie. Hennie zal hierna zoeken binnen de Catch+ context. De gevonden/gemaakte voorbeelden zullen worden gebruikt voor geautomatiseerde regressie testen.

3. SRU interface

Voor het zoeken zal een SRU interface op de server worden aangesloten. De RDF zal worden platgeslagen waarbij de Body en Target URI's geindexeerd worden zodat annotations voor specifieke targets direct gequeried kunnen worden. Bodies zijn in eerste instantie URI's.

De content van de Body zal dus opgehaald moeten worden, dit zal op een later tijdstip gaan gebeuren.

4. OAI interface

Er zal een simpele OAI interface worden aangesloten op de server. Het metadata prefix voor OpenAnnotation zal oac zijn. Er zal (nog) geen oai_dc ondersteund worden evenals dat er geen support zal zijn voor sets.

5. Known item search

dc:title, dc:created en dcterms:creator in aparte indexes zodat kan worden gezocht op title, creator en created.

6. Webform

Invoeren van een open annotation m.b.v. een simpel formulier

7. Webform zoeken

Simpele zoekbox met knop waarmee door de annotations kan worden gezocht.
