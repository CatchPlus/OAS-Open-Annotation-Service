Planformulier 4 - OpenAnnotation Service

1. Beheerder login

Er komt een login faciliteit voor een beheerder. De gebruikersnaam staat vast, namelijk 'admin'. Het password is bij default leeg. Dit zal bij de eerste poging om in te loggen leiden tot een scherm waarin het password gezet kan worden.

2. API Key beheren

De admin gebruiker heeft de mogelijkheid om een API key te maken. Aan deze API key wordt een username toegekend. Usernames zijn niet meer aan te passen zodra ze aangemaakt zijn. Tevens kan er een aantekening gemaakt worden voor wie de key is. We gaan kijken of er een best-practice manier is waarmee API keys gemaakt worden. Mocht er niks zijn dan vallen we terug op een UUID.

3. API Key gebruiken

Zowel het upload formulier in de portal als ook de SRU update moeten een API key aanleveren bij het plaatsen van records. De aangeleverde API key wordt door het systeem vergeleken met de uitgegeven keys. Als de aangeleverde key inderdaad gevonden wordt dan zal het record worden geaccepteerd. Voor queries hoeft geen key gebruikt te worden.

4. Meeindexeren bij upload

De username behorende bij de API key waaronder een record wordt toegevoegd wordt mee geindexeerd. Hierdoor kan er op een username gezocht worden wat hierdoor alle records oplevert die voor de desbetreffende username ingeschoten zijn. Dit is nodig om de admin user de mogelijkheid te geven alle records behorende bij een username te verwijderen.

5. API Key verwijderen

Door een API key te verwijderen worden ook alle records die met behulp van de API key zijn ingeschoten verwijderd uit het systeem. Om dit te laten functioneren is story '4. Meeindexeren bij upload' nodig
