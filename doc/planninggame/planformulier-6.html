1. Harvesting inbouwen

De OAS service wordt voorzien van de mogelijkheid om base url's te harvesten. Per base url wordt een apiKey gebruikt. Hierdoor kan gemakkelijk ook weer een geharveste repository worden verwijderd.

Los proces/service welke de config uitleest, harvest tot er geen resumptiontoken meer is.

2. Dashboard: Harvester management

Het admin panel wordt uitgebreid zodat baseurls en apiKeys geconfigureerd kunnen worden. Ook kan per repository aangegeven worden of deze geharvest moet worden of niet. Ook worden hier foutmeldingen getoond.

Het admin menu bevat nu een 'harvester dashboard' entry welke verwijst naar het scherm voor harvester instellingen.

3. Constrained Target

De constrained targets worden net als oac:Hasbody en oac:Annotation gepubliseerd. Hierdoor kunnen meerdere annotaties gaan verwijzen naar dezelfde contrained targets.

Voorbeeld: http://oas.dev.seecr.nl:8000/resolve/urn%3Aid%3Act%3A1

4. Zoeken op rdf:type

Er wordt een aparte index aangemaakt op rdf:type. Voor oac:Annotations houd dit in dat er een rdf:type gegenereerd zal moeten worden ter behoeve van indexering.

rdf:type="http://www.openannotation.org/ns/Annotation"
